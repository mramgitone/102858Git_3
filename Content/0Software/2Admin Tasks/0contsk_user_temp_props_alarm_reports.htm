The Alarm Reports Templates Dialog

Factory Template Properties (Alarm Reports) has the same fields as the User Template.

Name

Enter the name to be given to the Alarm Report. If a duplicate name is entered, it will be appended with a number to differentiate it from the original named report.

Alarm Type

Each Alarm Report requires an associated Alarm type and the system comes with pre-defined Alarm Report types. For a full list, refer to the Table of Alarm Detection Types.

From the drop-down menu, select the Alarm type. If an alarm is generated, you will see the associated, predefined icon displayed in the Graph.

Streams Tab

Include time series?

For stationary Alarms, the time series associated with the Alarm Report can be included as an additional option, with the Alarm Report. This time series is filtered to the same frequencies as the current analysis band. If the Alarm Report is viewed on a Panoptes server, the time series can be displayed and/or listened to. This is enables you to view and/or listen to the data that actually produced the Alarm Report.

Primary Required Stream

Each Alarm Report requires at least one required stream. To add a required stream:

  1. Select on the icon. From the drop-down menu, select the desired stream.
  2. Choose the Apply and Save buttons to create the streams.
  3. Repeat the process to add all the required streams.

To remove a stream if one or more of them exist:

  1. Select the icon next to the stream to be deleted.
  2. Select OK to confirm deletion. Choose Cancel to stop the deletion or Reset to restore the original values.
  3. Repeat the process to remove all the required streams.

If more than one required stream exists, the indicators of each stream will be combined, before it is entered into the Alarm Report. This is done with a Boolean AND operation.

Halos can be used to adjust the effective width of each stream prior to being combined with the Boolean AND operation.

Relative Halo (%) & Absolute Halo (m)

Relative Halos are used if more than one stream is included in an Alarm Report whereas, Absolute Halos are used to increase the original width of an indicator by the entered amount.

The Relative Halo is used to increase the indicator width, and a positive value will increase the indicator width by the entered percentage. Likewise, a positive Absolute Halo value increases the indicator width by the entered distance. If the values are set to zero, they will not affect the original width.

If a Relative and Absolute Halo is entered, the larger of the two will indicate by how much the indicator width will have increased.

Disallowed Stream

It is used if you require the exclusion of an indicator stream in an Alarm Report.

To add a disallowed stream:

  1. Select on the icon. From the drop-down menu, select the stream to be excluded.
  2. Choose the Apply and Save buttons to create the streams.
  3. Repeat the process to add all the required disallowed streams.

To remove a disallowed streams when one or more of them exist:

  1. Select the icon next to the disallowed stream to be deleted.
  2. Select OK to confirm deletion. Choose Cancel to stop the deletion or Reset to restore the original values.
  3. Repeat the process to remove all the required disallowed streams.

Halos can also be used to adjust the effective width of each disallowed stream prior to being combined with the Boolean AND operation. For more details and an example, refer to your training materials.

Tracking

History Length

It is used to track disturbances and provide information on the disturbance width, speed and acceleration.

The History length is the amount of time the system uses to decide if the alarm indicators in an alarm track have met the initial Tracking criteria, to generate a disturbance alarm. This field is used in conjunction with the Reporting Threshold to make this decision.

For situations where the system needs to alarm very quickly on short duration disturbances, it is recommended that you set the history length to a short time duration and/or set the Reporting Threshold to a small value. If set too large, both of these properties can significantly delay the actual Alarm Report.

In addition to being used for the Alarm criteria, the History length is the length of time that is remembered when attempts are made, to predict where an alarm track is likely to occur in the future.

History Weight Half-life

It is used to weight recent time samples more heavily if an alarm track is estimated. It is primarily used if you track disturbances whose motion does not strictly adhere to either the constant velocity or constant acceleration tracking models. This is particularly so, if longer History lengths are used.

To apply a uniform weight of the track estimate:

  1. Make sure the Uniform Weight check box is selected.
  2. Enter the desired History weight half-life (in seconds) to force the most recent time samples to bias the alarm track estimate.

For longer history lengths, it is advisable to set the History weight half-life to 25-50% of the history length.

Tracking Model

It is used to track Alarms that move. If it is known in advance that an Alarm will be move or be stationary, this information can be supplied to Helios. To do this:

  1. From the drop-down menu, select the Tracking model.
  2. For alarms select the option that best describes their acceleration speed. For example:
    • For those alarms that do not move (digging, leaks, and so forth), select the Stationary option.
    • For alarms that are not expected to accelerate (walking), select the Constant motion option.
    • For alarms expected to have significant acceleration (trains that pull in and out of a station), select the Accelerated motion option.
Alarm Reporting Interval

It is used to determine if a disturbance alarm can be generated at a rate equal to the FFT update rate.

You can send out Alarm Reports at a time interval between alarm reports, to the HWI display (or other 3rd party interfaces). The Alarm Report will be updated internally with the FFT update rate, but only sent out at the report interval.

Alarm Filters

Reporting Threshold

Alarm filters add another level of capability to ’ ability to alarm on valid disturbances and to reject false alarms and nuisance alarms. Any number of the Alarm filters can be enabled for a specific Alarm Report. An Alarm Report will only be generated if the disturbance passes ALL enabled Alarm filters.

This property is a fraction ranging between 0 and 1 and it MUST BE set so disturbance alarms can be generated. It works in conjunction with the History length to determine if the disturbance meets the Tracking criteria, and controls the generation of disturbance alarms. If this property is set to 1.0, every time sample in our history length must have an alarm Indicator before an Alarm Report is generated.

If a single time sample has does not include an Alarm Indicator, then no Alarm Report will be generated. If the Reporting Threshold is set to 0.5, then 50% of the time samples in the history length will require Alarm Indicators before an Alarm Report is generated.

For more details and an example, refer to your training materials.

Size Filter

Enables an Alarm Report to be passed, if its speed falls between the minimum and maximum speed in the currently selected units.

  1. To enable the Speed filter, make sure that its check box is checked.
  2. Enter in range values for the speed for an Alarm Report. An Alarm Report can only be generated if the disturbance falls within the specified speed range. The minimum and maximum speeds should both be positive values, as the filter only deals with absolute speed values.

One way to define a stationary disturbance, is to enter a minimum value that is slightly below zero, and a maximum value that is slightly above zero. If both the minimum and maximum speeds are set to zero, an Alarm Report will not be generated.

Burst Count Filter

A burst is defined as a group of indicators (in time) surrounded by a group of lulls. To enable the Burst count filter:

  1. Make sure the check box is selected.
  2. Enter values for the minimum and maximum burst count of an Alarm Report.

As an example, a walking disturbance will create a set of spaced bursts. Each burst corresponds to an individual foot plant.

If Burst filters are used, it is recommended that you set the History length to a longer time interval. This allows multiple bursts to occur within the History length.

Median Burst Period

This property defines the median time between the start of successive bursts. ONLY use it if the bursts from the disturbance are uniformly spaced in time. For example, constant walking produces reasonably constant time intervals.

To enable the Median burst period filter:

  1. Make sure that the check box is selected.
  2. Enter a minimum and maximum median burst period (in seconds) for an Alarm Report.

An Alarm Report will only be generated if the disturbance contains a median burst period that is larger than the minimum median burst period and less than the maximum median burst period.

Average Burst Density

The Average burst density is the average of the ratio of bursts to bursts plus lulls.

Example

A walking disturbance will produce bursts that correspond to foot plants, and lulls that correspond to the time between foot plants. If the bursts last on average for 0.5 seconds and the lulls last on average for 1 second, the Average burst density would be 33.3% or 0.5(0.5⁢+1)

To enable the Average burst density filter:

  1. Make sure that the check box is selected.
  2. Enter in range values for the minimum and maximum average burst density (in percentages), for an Alarm Report.
Activity Duration

This is the length of time that a disturbance lasts, and it can last longer than a History length period of time.

To enable the Activity duration filter:

  1. Make sure that its check box is selected.
  2. Enter values for the minimum and maximum activity duration (in seconds), for an Alarm Report.
Median Burst Duration

This is the median of the durations of ALL bursts that occur in a history length period of time.

To enable the Median Burst duration filter:

  1. Make sure that the check box is selected.
  2. Enter the values (in seconds) for the desired minimum and maximum median burst durations.

If you use the median rather than the average allows an anomalously high or low duration to still pass an Alarm Report, as long as the majority of the burst durations fall within the minimum and maximum values.

Median Lull Duration

This is the median of the durations of all lulls in a history length period of time.

To enable the Median Lull duration filter:

  1. Make sure that the check box is selected.
  2. Enter the required values (in seconds) in the minimum and maximum lull duration, for an Alarm Report.

An Alarm Report will only be generated if lulls last for a duration that is larger than the minimum lull duration and less than the lull maximum duration.

High Level Rules Tab

Report only if

This section adds another level of Alarm Report definition. After a disturbance event has passed the basic Tracking and relevant filtering criteria, it can then be passed through a set of High-level rules.

This feature assists in the generation of an accurate Alarm Report, which rejects false and nuisance alarms.

The first time this dialog is displayed, it will appear empty and you will see this text in the dialog: Report only if (No restrictions defined). To add a new rule:

  1. Select the icon. A new rule template is displayed in the dialog as shown below.

New Rule Template displayed in the High Level Rules dialog

  1. From the drop-down menu, select the Alarm Report for the newly added rule.
  2. Select whether this Alarm Report occurs or does not occur within the selected distance and/or period of time.
  3. Enter in the distance separation and time interval that the selected Alarm Report will or will not be active in.
  4. Select the Apply and Save buttons to create an Alarm Report.
  5. Repeat this process to add additional High-level rules.

To remove a High-level rule:

  1. Select the icon beside the rule to be removed.
  2. Select OK to confirm deletion of the rule. To stop the rule deletion, choose Cancel or to restore the original rule, select Reset.
