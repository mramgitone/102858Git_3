Check and Display the Shot

You MUST BE logged in as either an Administrator or Factory User to change this setting but when a user is logged in with View Only access, they will only see the option that has been configured to be viewed. This view may need to be customised to suit the three types of user.

For the system to display the raw signal that comes off the optics module when the system starts, the Display Type needs to be configured to show the Raw Shot.

The default scale for the Raw Shot display type is 32767. This is the maximum value for raw data. Even though the Input voltage was set at the factory, there can be data values in a zone of interest that are clipped (they continuously display at the maximum value). In these circumstances, it is recommended that you increase the Input Voltage Range to prevent those locations from being clipped.

Complete these steps:

  1. In Currently Viewing > Display, select the Raw Shot option, to check the right optical settings are configured. The other options to choose are shown in Table of Display Types.

The four Shot types do not require an active Alarm Report in order for them to display in the Graph. An Alarm Report is made active if it is selected in the Alarm Report tab of the Optical Channel Properties dialog. Select the desired Shot type for it to display in the Graph.

  1. Start the System, refer to Change the Monitoring Status. The Controls dialog will be displayed. The CPU temperature can vary dependent upon each installation.

The Controls Dialog

  1. To change the Input voltage range, select the Input Voltage drop-down menu and select the next higher value. Select the Apply and Save button for the change to be confirmed. This will momentarily stop the system and restart it with the new Input voltage range.

  1. It is recommended that the sensor end position be displayed. To do this, make sure that the distance that corresponds to the end of the sensor is in agreement with the known sensor distance. Distance discrepancies are typically due to errors in the RI setting being used.

It is important not to clip the raw data, as this will introduce non-linear distortion into the processing system. This can reduce significantly the sensitivity of the portion of the sensor being clipped.

  1. It can be necessary to determine the movement velocity of an event disturbance on the Graph display. Velocity tracking is the default mode of the Graph display. For instructions about how to create and remove a Velocity, refer to Velocities.
