Detection Statistic

Analysis band low & Analysis band high

This restricts the analysis to the frequency range that contains the disturbance. A significant increase in the SNR of the system improves the ability to alarm the required disturbance. Since the majority of the environmental noise and nuisance energy contains lower frequencies, keep Analysis band low field as high as possible.

The Analysis band high field MUST BE set to the high end of the frequency range. This isolates the disturbance signal that is detected. A lowered Analysis band high cut frequency can assist in the reduction of the compute load.

Power Spectrum Averaging

PSD half-life

This is used in conjunction with the Detection Statistic half-life to regain time resolution by the reversal of temporal smears from the PSD Kalman filter. Careful management of both Kalman filters, improves SNR values without the negative effects of the temporal smears. The longer the PSD half-life, the more smears are applied.

For weak, persistent disturbances, such as leaks, a typical value could be as large as 10 to 15 seconds. This can reduce the noise floor, and allow lower amplitude signals to be seen above the noise floor.

To apply temporal smears, enter a non-zero value for the PSD half-life.

If the PSD Half-life is too long, it can remove short-time duration impulsive disturbances. ONLY use it with low-amplitude persistent disturbance

Detection statistic half-life

Governs the time dynamics of the system response, and works in conjunction with the PSD half-life.

This property is designed to tighten up (de-smear) the time resolution that the PSD half-life tends to remove (smear) as it attempts to pull weak signals above the noise floor.

The Detection statistic half-life is set to a lower value than the PSD half-life, to sharpen up (de-smear) the time resolution of the detection statistic output. This is because the noise is random, whereas the signal is more coherent over time.

WCS width

The width of the spatial filter is selected from the values, represented as odd numbers, from the drop-down menu. The values represent the total width of the filter.

  • Make sure that the Enable spatial averaging check box has been selected.

Both the check box and the WCS width field can be adjusted dynamically.

If the WCS width is larger, it will produce more spatial averaging and if set too large, it can smear disturbances that appear close together. If the WCS width is smaller, it will produce less spatial averaging. In addition, the location accuracy of a disturbance tends to decrease as the WCS width is increased.

Signal Separation

Enable strong/weak separation

This module is used when there are reasonably strong signals relative to the background noise. In the case of footsteps, digging or vehicle movement, this is the most common approach to take. Strong amplitude implies that the signal is sufficiently high, relative to the noise floor, to not require a lot of temporal averaging to detect it.

If this module is selected, the Strong/weak separation factor text field is enabled, as well as the check boxes to include the strong impulsive spectrum and/or the noise floor spectrum.

Strong/weak separation factor

This is a positive number greater than or equal to zero. The smaller this value is, the more noise energy will make it into the Strong Signal component.

If the Strong/Weak separation is enabled, and the separation factor is set to 0, you will see the noise level grow slightly. This is due to the way the Detection Statistic is computed.

If it moves from 0 to 1, the noise level is reduced. As this value gets larger, there will be more noise removed from the strong signal. A typical value is around 2 to 5.

In order to separate out the noise from the strong signal, make sure the Include Strong Impulsive spectrum option is selected, and that you de-select the Include Noise Floor spectrum option. If all the available spectra are selected, then there will be no separation.

Enable temporal decomposition

This routine is designed to split out signals that exhibit impulsive temporal from more persistent characteristics. Footsteps would be an example of an impulsive signal, whereas a leak, or even a vehicle driving along the would be an example of a more persistent signal.

The smaller the Baseline Doubling Period and the Baseline Recovery Period values, the more impulsive energy can leak into the persistent signal spectrum.

Baseline doubling period & Baseline recovery period

These properties are used to define the characteristic time scale of a disturbance you consider to be a persistent disturbance. For example, the walk detection alarm can be configured so that each foot plant lasts about 0.5 seconds.

The values relate to how quickly the tracked function can react to changes in the energy response.

Make sure the Baseline periods are set. This enables the tracked function to stay below the impulsive component, and manages the persistent component.

Preconditioning half life

This property governs the length of a temporal smoothing filter used to minimize the effect of temporal amplitude variations, due to variations of sensitivity in the detectors used.

This value should be set between 0.02 to 0.1 seconds, to ensure that persistent signals do not get put into the impulsive component.

Enable weak/noise separation

This routine is used when the signal that is detected is quite weak relative to the noise floor.

This method uses the same separation algorithm as the Strong/Weak separation, but does it after it calculates the average time (with the use of a Kalman Filter). This isolates the persistent weak signals from the pseudo random noise floor. It also improves the delectability of weak persistent signals such as low energy leaks.

Weak/noise separation filter

The check box MUST BE to enable the Weak/Noise Separation module. If enabled, the Include weak signal spectrum and/or the Include noise floor spectrum check boxes will be active.

The value is a positive number greater than or equal to zero. The smaller this value is, the more noise energy will make it into the weak signal. If it is set to zero, it will put all the noise energy into the weak signal component of the power spectrum.

If the value is moved from 0 to 1, it will reduce slightly, the noise level.

As this value gets larger, there will be more noise removed from the weak signal. A typical value is around 2 to 5.

Spatial Processing

Enable spatial amplitude adjustment

The Spatial Amplitude Adjustment module is useful to fill in spatial gaps in our Detection Statistic. This module attempts to interpolate a continuous response over the width of the event disturbance to remove these gaps.

The objective is to differentiate disturbances that are localised to a short length of (local), from those that affect much longer sections of the sensor (non-local).

Structure element width

This field attempts to smooth out spatial amplitude variations. If this value is set to zero, it applies no spatial smoothing. If the value is increased, the amount of spatial smoothing is increased. Set this width to a value that is less than or equal to the current Pulse Width. For more information about the Pulse Width, refer to Section XXXX.

Typical values for the Structure Element Width are between 2.5 and 7.5 metres.

If this value is too large, it will produce poor results. If you see large triangles in the event detection value, this indicates there are artefacts that have too much spatial smoothing.

Enable spatial decomposition

This distinguishes between and separates, local, spatial event disturbances from non-spatially local event disturbances. Make sure this check box is selected to enable the Spatial cutoff size field.

You can exclude the local spectrum from non-local spectrum. To separate these signals, make sure that only one of the spectra is selected.

Spatial cutoff size

This sets the boundary between disturbances that are deemed to be local and non-local. Values are set to zero for non-local components and greater than zero for local components.

Spectral Shape Analysis

Enable spectral shape analysis

This separates the signal energy into three different spectral shape components:

  • Low Frequency spectrum (modeled)
  • Low Frequency spectrum (un-modeled)
  • Non-low Frequency spectrum

This enables narrowband signals that are buried completely in the low frequency modelled spectrum, to be resolved.

Make sure that this check box is selected to enable the Activation threshold. In addition, the options to exclude the low frequency spectrum (modelled), the low frequency spectrum (unmodelled) and the non-low frequency spectrum are also available. In order to successfully separate out these three spectra, they MUST BE deselected.

Activation threshold

This value is used to define the spectral data used as input into a curve fit. This will define a boundary between the expected (modelled) low frequency spectrum, and the non-low frequency spectrum.

Typically, for narrow band disturbances, you need to discard the low frequency modelled spectrum, and keep the other two spectra. Typical values for the Activation threshold are between 2 and 4. The value is dependent on the amplitude of the low frequency modelled spectrum.
