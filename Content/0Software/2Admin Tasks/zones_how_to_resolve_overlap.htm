How to resolve an Overlap

This procedure is best explained by the use of an example.

Example

Let’s say you need a new zone, labelled as Zone2, which extends from 300 m to 550 m. When Zone2 is created with those distances, it overlaps with Zone1. If there has been an overlap, once the Submit button is selected, a warning message is displayed. The image below states that Zone2 overlaps Zone1.

The Overlap Warning Message

To resolve this:

  1. Select the option Truncate Self. This shifts the right edge of Zone2 to the left to remove the overlap.

OR

Select Truncate Sibling(s) to shift the left edge of Zone1 to the right. The image below shows how the zones look if Truncate Sibling(s) is selected. The left edge of Zone1 was changed from 300 m to 350 m in order to remove the overlap.

The Truncate Sibling(s) Dialog

  1. Define a New Zone, called Zone3. For instructions. Make sure the new zone extends from 350m to 500m. Zone3 is located entirely within Zone2. A warning message is displayed to indicate that Zone3 is within another zone, as shown below.

Warning Message to Split Sibling within a Zone

  1. Select Cancel to stop the creation of the new zone, or Split Sibling, to create an additional zone, called Zone4. This will appear to the right of Zone 3. The image below shows the zones after the Split Sibling button is chosen. As shown, Zone3 is inserted into Zone2, and the right side of Zone2 is replaced with Zone4. See the image below.

Dialog that displays the created zone

If a new zone is created to rectify an overlap issue, it will inherit all of the properties of the zone that it replaces. In addition, if the new zone completely encompasses its siblings, there is an option to delete the sibling zone all together.
