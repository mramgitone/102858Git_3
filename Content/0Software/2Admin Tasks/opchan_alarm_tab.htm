Alarm Report Tab

The Alarm Reports tab controls defines which Alarm Reports are generated by the system, see the image below.

The Alarm Report Tab

Alarm Reports are then submitted to the clients, and optionally to Panoptes servers or 3rd party interfaces. controllers are typically shipped with at least one pre-configured Alarm Report. However, streams need to be created and associated with an Alarm Report, to activate the Alarm Reports option. For more information about how to create a stream, refer to your training materials and Streams Properties for more information.

If you have a Panoptes server connected to your system, refer also to Reporting.

An Alarm Report can be created, even if you do not want to generate and submit an Alarm Report for that stream. During the initial property tuning stage, it is also common to create an Alarm Report but not submit it. If these types of Alarm Reports are created, the system will not be flooded with reports.

To have multiple active Alarm Reports implies that multiple streams are processed in parallel. This can increase the compute load. Care MUST BE taken to activate the Alarm Reports, one at a time, to monitor the increase in compute load and prevent a system overload. If necessary, perform some data decimation.

Refer to the Spatial decimation factor for more information about how to reduce the compute load on the Helios system.

Below are instructions to create, activate and remove Alarm Reports. When this section has been completed, refer to Hardware Properties to set-up the initial properties for the System.

You will need to create at least one Alarm Report and activate it. Click or tap here for more details.

Create an Alarm Report

An Alarm Detection determines if the response is from a desired disturbance, and if so generates the appropriate Alarm Report. Additional logic performs most of the actual disturbance classification and nuisance rejection work. For the system to generate an Alarm Report, you need to make sure an Alarm Report is selected. To do this, one or more Alarm Reports, with the associated streams, detection statistic templates and alarm cleaning templates, must exist. Most Helios systems will be shipped with a set of non-editable factory templates that can be used to start alarm generation.

  1. To create an Alarm Report, you need to check the list of stream templates, and if you need to create a new Alarm Type, refer to Streams Properties for instructions.
  2. Then check and configure the settings to be shown in the Alarm Report. Refer to Detection Statistics for more information.
  3. To activate or remove an Alarm Report, refer to How to activate an Alarm Report and How to remove Alarm Reports respectively.

How to Activate an Alarm Report

Alarm Reports are listed by name. When an Alarm Report is created, it must be made active. To do this:

  1. Select the check box next to the Alarm report to be submitted.
  2. Select the Submit alarms check box. This configures the system to generate and submit the Alarm Report.
  3. To make it easier to submit Alarm Reports, make sure the Submit alarms check box is selected.
  4. Enable each of the Alarm Reports that the system has configured to report on. To do this repeat Steps 1 to 3. To remove an Alarm Report, refer to How to remove Alarm Reports.

How to suppress Alarm Reports

During system tuning, you can get many false alarms in your alarm reports. You can suppress the submission of these false alarms to the rest of your engineering system, which can also speed up the . To do this see the image below and complete these steps:

The Alarm Report Tab

  1. Check that the Submit alarms option is not checked. This allows you to create alarm types and monitor their stream detection and indicator display values, whilst not flooding the system.
  2. Once you have tuned the system, select Submit alarms to turn alarm detection back on again.

How to remove Alarm Reports

There are two ways to remove Alarm Reports: 

  • Select each individual Alarm Report and then deselect the check box for each Alarm Report.
  • Select the Clear All link at the bottom of the Alarm Report tab, to clear all active Alarm Reports.
