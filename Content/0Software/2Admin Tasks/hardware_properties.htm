Reporting

The properties contained in the Reporting tab, shown below, are only used to communicate with a Panoptes server, and/or a third party interface. It is split into two sections: XML submission type and Binary submission type.

The XML submission type holds information that the Helios Controller sends out to third party interface in an ASCII XML format to specific URLs.

The Binary submission type entries are only required when a Panoptes server is connected to the system. These options send disturbance alarms and Graph data to the Panoptes server.

The Reporting Module displayed in the Hardware Properties dialog

XML Submission Type

  1. Check the Time Zone that you want the XML message timestamps to be displayed in.
  2. Select the appropriate Submit disturbance alarms and Submit system alarms checkbox. A URL must be specified for the communication to be established with the server that will receive this information.
  3. Select if appropriate, the Submit heartbeats checkbox. A URL must be specified for the communication to be established with the server that will receive this information. A heartbeat rate must be specified to determine how often the heartbeat is sent.

Binary Submission Type

  1. In the Binary host field, enter the URL details for the Panoptes server. This is an 11 digit number.
  2. In the port field, enter the server port number. These properties are a factory setting and is not changed unless there is a non-standard communication requirement. If the latter is a requirement, or you experience issues with the communication to the Panoptes server, refer to the Panoptes User Guide, and contact Customer Support.
  3. Select the Submit disturbance alarms checkbox, so alarm detections can communicate with the Panoptes server.
  4. Select the Submit shots checkbox to enable Graph data to be sent to the Panoptes server.
