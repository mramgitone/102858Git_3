User Template Properties (Indicators)

Below is an image of the Indicator User Template and a description of the fields for each section.

Indicator

Name

Enter the name to be given to the Indicator Cleaning Mask. If a name is duplicated, the name will be appended with a number to differentiate it from the original mask.

Indicator Cleanup

Enable morphological cleaning?

This check box has to be selected to enable indicator cleaning and edit any of the properties in the Indicator Cleaning dialog. If it is not selected, none of these properties can be edited and indicator cleaning will not be performed.

The No Morph Clean Factory Indicator Mask does not perform any indicator cleaning at all. This mask only creates a stream that passes raw alarm indicators into the Alarm Reporting module.

Maximum false negative rate

This property assists in the removal of false negatives due to spatial sensitivity fluctuations in the system response. It modifies the logic to only remove runs of connected alarm indicators that are less than the product of Maximum false negative rate and Minimum alarm size.

Use of this property can reduce the likelihood that a connected sequence (run) of alarm indicators will be removed because it fails the Minimum alarm size filter.

A reduction in value will remove more false negative values, while it increases the value and removes fewer false negative values. If the Maximum false negative rate is set to 1.0, it will not remove any false negative values.

When a WCS spatial averaging filter or Spatial Amplitude Adjustment is used, the number of false negatives is significantly reduced, which reduces the need to use this property.

Minimum alarm size

If a false positive is found, the alarm indicators associated with it are discarded. If we know that the disturbance of interest is likely to be activate at least 5 metres of the sensor, then we can set the Minimum alarm size to be 5 metres. Therefore, any disturbance of less than 5 metres could not be from the disturbance of interest.

If the Minimum alarm size is set too high, it can remove valid indicators that just didn’t meet the Minimum event size criteria.

Minimum lull size

This parameter attempts to remove any false negatives. A false negative is measured in metres, and is any gap in an indicator run that is less than the Minimum lull size. When a false negative is found, the gap is filled in with alarm indicators. The clean routine assumes that if the occasional location has missed an indicator, this is because of the spatial sensitivity fluctuations, and it simply fills it in.

Maximum alarm size

This property is measured in metres, and is used to reject disturbances that are wider than the Maximum alarm size. This check occurs after false positives and false negatives have been removed.

Its purpose is to reject disturbances that are much wider than the disturbances of interest.

In certain circumstances, the Maximum event size property can remove the alarm indicators for two disturbances that occur over the same location range. Use with caution as this can sometimes remove the disturbances you are trying to alarm on. It is not typically used any more, as it is safer to reject wide disturbances with the Alarm Size filter. Refer to Alarm Size Filter.

Minimum alarm duration

This repeats the spatial indicator clean on the neighboured time samples for a given position on the .

The temporal indicator clean attempts to remove any false positives in the temporal dimension. A temporal false positive is measured in seconds. It is any temporal run of connected alarm indicators that is less than the Minimum event duration.

Minimum lull duration

This attempts to remove any temporal false negatives. A temporal false positive is any gap that is less than the Minimum lull duration in seconds.

When the temporal indicator clean properties are set, it is recommended that every detection statistic setting, is displayed. The Graph update rate MUST BE set equal to the FFT update rate. This may not be possible in LiveFibre mode, due to the length of the being monitored.

If a representative time interval of raw data is logged and replayed in the Playback module, you can set the Graph update rate to the FFT update rate. This allows all the indicators to be displayed in the Graph. This makes optimization of the indicator clean properties much easier.

Make sure the Playback speed is set to a small number such as 0.1.

You can also reduce the size of the display browser window to increase the update rate.
