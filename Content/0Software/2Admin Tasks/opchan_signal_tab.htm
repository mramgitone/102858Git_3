Break Check

The Break check module is designed to trigger an alarm when the sensor is severed or damaged. The Break scans the response, and identifies the break detection value along the length of the sensor. Large amplitude discontinuities occur as normal on a sensor, and can be similar in magnitude to a break. This makes a break harder to detect. However, if there is a clear distinction between the sensor discontinuity and sensor break, then it is easier to specify a break alarm threshold. When this threshold is breached, it will generate an alarm that indicates there has been a break.

A problem occurs when there is not a clean boundary between the magnitudes of the break detection value between normal sensors and broken sensors. On screen, you will see a break represented, as shown below.

When the mean values are not separated by the sum of the half-error bars, there is overlap in the distribution curves, and now, regardless of what value the break alarm threshold is set to, there will be a possibility of either false positives (detecting a break in a normal fibre sensor) or false negatives (failing to detect a valid fibre sensor break).
