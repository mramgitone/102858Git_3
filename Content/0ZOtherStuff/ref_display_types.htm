Table of Display Types

Display Type Description
Default The Default display type has been designed to respond to any valid signal disturbance that occurs between DC Cutoff and Nyquist (the full frequency range as defined by the PRF). It does not require an active Detection Statistic from an active Alarm Report. The advantage of the Default display type is that it shows the sensor’s response to disturbances as a function of distance without an active Alarm Report.
Raw Shot This displays the raw digitized data that comes in directly from the optics module, as a function of distance. The raw data contains values between zero (0) and 32,7687. In addition, all of the previous data display types are derived from the power spectra of the input signal, and all have an update rate equal to the FFT update rate. The raw data comes in at the PRF, which is at a much higher update rate. When the raw shot is displayed, the system simply takes the first shot of every FFT block, and sends that raw shot to the display. No temporal averages occur in the data reduction.
Mean Shot The Mean Shot display shows the mean value (or DC component) of the response at each location over the FFT interval. It is very similar to the Raw Signal display, but updates much slower (at the FFT update rate rather than the PRF). This display type shows variations of the non-time varying response (over the duration of an FFT block) as a function of distance. This response closely reflects the ambient response of the sensor at each sample point.
Mean Shot Drift

This displays the amplitude variation of the Mean Shot (also known as the DC Baseline) between FFT blocks as a function of distance. Ideally, there should be very little drift over time, as the DC Baseline is the response in the absence of any physical excitation.

In practice, the Mean Shot drifts a fair amount from FFT block to FFT block. The Mean Shot Drift provides an indication as to the magnitude of this temporal drift.
Signal RMS

This displays the time series Root Mean Square (RMS) of the time samples, over an FFT block due to signal power only, with the band limited to the analysis band.

If you are a Factory User, refer to the User Template Properties (Detection Statistics) for a description of the analysis band.

Noise RMS This is very similar to the Signal RMS value except that it is the time series RMS over an FFT block due to noise power only, band limited to the analysis band.
Total RMS This displays the time series RMS over an FFT block that includes signal and noise power, with the band limited to the analysis band.
Stream Detection Value

This displays the Detection Statistic for a particular stream. This value is a pseudo Signal to Noise Ratio (SNR) of the response of the , with the band limited to the analysis band. This value is computed in two stages:

  • The raw Detection Statistic is computed if the spectral processing properties is assigned to a particular stream.
  • The raw Detection Statistic is optionally de-smeared if the Detection Statistic Half-life assigned to a particular stream. The Stream Detection Value is used as input for the Alarm Threshold module, which generates the Stream Indicators.
Stream Indicators

This display is used a lot. As described above in Stream Detection Value, raw indicators exist when the Alarm Detection Value falls within a user-defined amplitude range. All time and space points with Alarm Detection Values above the Alarm detection threshold, and below the Alarm suppression threshold, will produce a raw indicator. These raw indicators are can be cleaned to remove false positive and false negative values.

Cleaned indicators are displayed with this data display type as it optimizes the Alarm detection threshold, and the Indicator Cleaning properties, to produce a solid stream of cleaned indicators. These appear where disturbances of interest are located.

Stream Indicators are the input into the Alarm Reporting module, and without properly conditioned indicators, the probability of robust alarms on desired disturbances is greatly diminished.

Alarm Indicators

An Alarm Report can consist of multiple streams, each with a set of Stream Indicators associated with it.

This display type shows the Boolean combination of the cleaned indicators which are used as input into the Alarm Reporting module. This is very helpful if you have to fine-tune the Boolean logic, used if you combine individual streams being fed into the Alarm Report logic.
