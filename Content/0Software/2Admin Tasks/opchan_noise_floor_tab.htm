Noise Floor Bounding Tab

In order to produce a signal/noise value (S/N value), the Helios system will automatically compute a noise floor value to use as N. This is performed automatically as it is assumed that noise will generally cover a large frequency range, while the signal will have a more limited frequency range. However there are some environments, where the signal covers a large frequency range, hence it is interpreted as noise and taken out of our S/N displays. It is in these applications that the bounded noise floor becomes useful.

The Noise Floor Bounding Tab

When to use a bounded noise floor

This depends on your application. The standard, unbounded, noise floor works usually does not need to be checked once it is first set. The time to use the bounded noise floor is when you have an application where the signal covers a large frequency range, and is interpreted by our system as noise. If the noise floor needs to be changed, there are two symptoms that the system will display:

  • Within the Default or Stream Detection Value, the S/N field does not display a signal.
  • There is a lowered signal where you expect to see an increase.

Check the Noise Floor

To check the Noise Floor is displayed correctly, open the Noise Floor Bounding Tab. See the The Noise Floor Bounding Tab image above.

This dialog displays the typical exponential decay along the length of the . If it does not, that is another indication of signal interpretation as noise. For example, an increase can occur at times that you expect to see more signal.

If you see either of these cases, you should consider the use of a bounded noise floor.

How to use a bounded noise floor

Once you have decided that you wish to use a bounded noise floor, you need to configure the system to use it. Complete these steps:

It is necessary that a quiet is used for this procedure.

  1. Select the Optical Properties Channel > Noise Floor Bounding tabs. See the The Noise Floor Bounding Tab image above.
  1. Run the live system with a and start to configure the noise floor bounding controls, to get the noise floor bounding curve, which is displayed as a line graph, like the one displayed below.

The goal will be to get the noise floor bounding curve (which is displayed as a line graph on top of the noise floor bounds display) to be just slightly above the level of the noise floor bounds display.

Enable Noise Floor Bounding

If the Noise Floor Bound has not been configured, then contact us. A Fotech Engineer will need to configure and define the portion of the noise floor. Refer to Calibration of the Noise Floor.

This is where the noise floor bounding can be turned on or off.

After the Noise Floor Bounding has been turned on, look at the noise floor display. If the bounding curve appears exponential in its shape and seems to sit just slightly above the bulk of the noise floor, then this is all you need to do. If not, there are two other controls that you need to adjust: Bounds offset % and Use exponential fit .

Bounds offset %

If the bounds curve seems to have the correct shape but is either too high above, or is below, the noise floor, then it can be moved either up or down with this control.

Use exponential fit

It is only necessary to use this option if the noise floor model does not produce a good curve on its own. This option is only used with the Bounds Offset (%) setting to produce a usable noise floor bounds curve.

Most of the time the noise floor bounds curve should automatically look like an exponential curve. If the noise floor is not exponential, the will need to have a positive offset in order to be useful. To do this:

  1. Select the Optics Tab and check the settings. Make sure the optical properties are suitable for the application.
  2. If the curve has still not rapidly increased in size, you can force it to be more curved. Turn on the Use exponential fit option. This will often need to be used in combination with the Bounds offset % setting.
