Change the Properties of a Zone

Zones can have a number of their processing properties customised. By default, when created, a zone inherits the global sensor properties. There are many situations when overrides to the global processing properties can occur. In addition, individual zones may be fine-tuned to respond to specific disturbances.

To change the properties of a zone:

  1. Select the icon. It is to the left of the icon. This brings up the Zone Properties dialog. This dialog contains the Detection Statistic properties that can be overridden on a zone-by-zone basis.

<image with callouts - The first section contains the basic zone properties, including the zone name and the start and end distance for the zone. These properties are independent of any individual stream.>

  1. Helios is a parallel stream system. If you want to override the remainder of the zone properties, the changes must be associated with an individual stream. To associate a Stream with any property changes, select the Stream name from the Stream drop-down menu.

If there are no streams associated with the zone’s channel, then the zone properties cannot be overridden, and you will see the properties dialog shown above. However, you can change the properties for an inactive stream. Refer to Change Inactive Stream Properties.

  1. In addition, certain properties can be adjusted in the global property dialog, and can be overridden for more than one stream. A common use of this feature is to set a unique Alarm Detection Threshold for different streams. Refer to How to set an Alarm Detection Threshold for Streams.
