Wipe Database

Precautions

Before you wipe the database, check with the person in authority that this should be done and then do a back-up of your system.

Once you wipe the database, the information cannot be retrieved. This should only be done as a last resort, typically on the instructions of a Representative.

Once a system has been wiped you have a truly empty database and will have to enter ALL content from scratch, manually. In particular you will need to reset the various hardware and optical settings from the original specification sheets.

Instructions to Wipe Database

A wipe is the last resort for a troubled system and you will have to enter all data from scratch. If you try to restore the wiped database, you can reimport the problem you are trying to fix.

  1. Select the Factory Menu > Wipe database option and a warning message will prompt you whether you wish to proceed or not. Select OK. Can I actually have a screenshot?
  2. Before the database is wiped you will be given a warning and a chance to change your mind.
  3. A snapshot of the database will be automatically created which can be restored if required and is stored as "Backup from last system wipe" in the Manage Snapshots dialog.
  1. After the wipe is completed the system will automatically restart and you will be presented with a truly empty database.
  2. Populate all the Hardware Properties and Optical Channel Properties dialogs before you use the system.

Save Factory Defaults

There may be times when the edited properties no longer produce acceptable results in the Graph. If this happens, the Helios system can be restored to its Factory Defaults.

This process needs to be completed before the newly configured system is sent to the customer. Click or tap here to see below.

Use this function with extreme caution. If a Helios Controller is reset to its factory defaults, all property changes made to the system will be destroyed! This includes all zone definitions and their associated properties.

  1. Any testing settings should be removed, the hardware settings should be checked to make sure they are correct, the proper serial numbers entered, a unique UUID created, and all available channels for MUX systems created.
  2. Any customer bespoke configurations should be completed.
  3. Select Save as factory defaults and this will save the settings in a special location which will prompt the Admin/Reset to factory defaults menu item to appear, where you would:
    1. Select Admin > Reset to factory defaults. A confirmation dialog will be displayed.
    2. Select OK to reset the Helios system to its Factory Defaults. The system will then shut down and restart the Helios Controller.
    3. Select Cancel to dismiss the warning message and return to the Operations Screen.

Temporary System Thresholds

The Temporary System Thresholds dialog is where the system temperature and disk space allowance can be controlled and tested. The main reason to run these tests is to force an XML notification of a system alarm so you can test the high temperature and the low disk space warning system messages. These tests can be conducted together or as lone incidences.

Instructions for Temporary System Threshold Tests

The logging drive tests are only performed if the system is actually logging. If the logging drive is full but we are not logging, then no warning is given.

  1. Select Factory > Temporary system thresholds... and the dialog below is displayed. Enter the required details, as shown in the image below.

Temporary System Thresholds Dialog

  1. Set the values in the Admin > Hardware Properties > Reporting tab.
  2. Open the System Health panel .
  3. Test the System Temperature Warning. To do this, open the Temporary System Thresholds dialog and enter a value one or two degrees lower than the current temperature.
  4. Test the Disk Space Warning thresholds. To do this enter a value for the threshold beneath the System Drive and Logging Drive values.
  5. Select Submit to run the test and Cancel to stop the test. The CPU Temperature will turn amber and our blue spinning warning icon appears.

The System Threshold Test Results

  1. For the CPU Temperature test, lower the threshold a further 5°C below the CPU Temperature value and a warning message will appear in the status message area.

  1. Lower it another 10°C below the value, and the system will display a temperature error pop-up message. This will automatically stop FDEL to allow the CPU and GPU to cool down.
  2. The tests are now complete. When you press the Cancel button, close the dialog, or refresh the page the system, the temporary thresholds and the alarms and warnings are removed automatically and cleared.

Microcontroller console...

This process should only be completed by trained personnel. Take care when you enter data because the vendor cannot be held liable for any errors made!

When chosen, the microcontroller console opens in a new window. This console allows you to enter (manually) the commands to be sent directly to the optics unit microcontroller.

This console is intended as a troubleshooting aide, as well as to test proof of concept hardware. The interface is simple and intuitive to use. When it first comes up it will be connected to the necessary WebSocket port of the Helios unit and has the functions shown in the image below.

The Microcontroller Console

Simply enter the word “help” to get a list of the commands that the current hardware supports, or enter the file format *.help, to get some on-screen instructions about how the terminal window itself works.
