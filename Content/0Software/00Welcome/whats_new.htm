What’s New

Version has added three significantly new features which may encourage you to upgrade your software:

  • Bounded Noise Floor.
  • Overlapping FFTs.
  • HDF5 & NetCDF4 Logging Formats.
See below for an explanation of each feature.

Bounded Noise Floor

While the standard noise floor estimate computed by a Helios unit has worked well for many installations, there are a few types of signals, namely signals that have a wide frequency range that can confuse the system. With version we have added the concept of a "bounded" noise floor that significantly improves the response in these situations. Refer to Noise Floor Bounding Tab.

Overlapping FFTs

In previous versions of our software the user has had to give up temporal resolution in order to gain frequency resolution. That is, increasing the FFT size has always resulted in a loss of precision in time. With version we have added the ability to use an overlapping FFT in order to gain frequency resolution without having to give up the temporal resolution. Refer to the Processing Tab.

HDF5 & NetCDF4 Logging Formats

Historically when recording raw data from a Helios system you have had to record to our FDS (Fotech Data Stream) format. This format was optimized to allow the fast writing required by our system, and for a long time was the only option we offered. With version we have added the ability to log to the HDF5 and NetCDF4 formats. Removed/Deprecated Features

There are two features that have been removed from version of the Helios software: the 180 MSPS Sample Rate and Older 4U Hardware.

See below for an explanation as to why these have been removed.

180 MSPS Sample Rate

This applies only to older optics units. Newer optics units support only the 150 MSPS sample rate, which is the most accurate (and has the least optical "jitter") of the sample rates. The 180 MSPS sample rate has always been problematic with some of our digitizing cards giving a good result and others giving poor results. In the past we have simply relied on the end user knowing whether or not their card would get a good result and not using this option if it does not.

There is no way to tell from the card itself. The only way to recognise whether a digitizing card yields a good result is to log raw data and examine the results.

However, there have been enough reports of problems with logged data being sub-standard due to the end user not realizing that this setting would give a poor result on their system. Hence it was decided to remove this optional altogether.

Older 4U Hardware

Support for the older hardware (4U boxes) ceases with this new release.

If an older box is in operation, an error message will be displayed when version is installed onto the hardware. To use the new features in version of the software, a hardware upgrade is required. Contact us for more information.

The older hardware will continue to support V4.5.X via any critical patches that are deemed necessary.
