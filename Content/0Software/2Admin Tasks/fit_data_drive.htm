Data Logging & Playback Controls

How to fit a Data Drive

Data logging management is located to the right of your display and allows the user to log to an optimized internal data file, known as an FDS. To do this, the system records raw and graph data over the entire monitored section, which can be used in Playback mode. The user can also enter a defined subset of the entire section, or over one or more pre-defined zones.

These controls let you control how you mount a disk drive and start data logging. To do this, go to the relevant section below and complete the tasks. The disk drive has to be mounted before logging can begin.

  1. To mount a data drive, select the drive-mounting icon . This will display the dialog below.

The Drive Mount Utility Dialog

  1. Select the required disk drive from the list shown. Only use the No external mount option if you know you have enough internal space on the data drive, or if you do not have access to an external data drive.
  2. Define the directory and file name.
  3. Select the Submit button to confirm, or the Cancel button to select an alternative disk drive or stop the selection process.

Unmount a Data Drive

  1. To unmount a data drive, select the unmount drive icon . This will display the dialog below.

Unmount Device Prompt Message

  1. Select OK to confirm removal of this device drive or Cancel to leave the drive mounted.

Start Data Logging

The Data Logging Properties must be checked when the system is not logging data. These properties can be viewed in the Data Logging Controls dialog.

Data Logging Controls dialog

If data is being logged, and the distances are changed, the current FDS file will be closed and a new FDS file will be created with the new distances displayed. The Data Logging Properties can be checked in two ways:

  • Select the icon to access the Data Logging tab.
  • Select the to access the Dynamic Data Logging dialog.

Data Logging

See the image below and complete these tasks:

The Data Logging Tab

  1. Locate the directory to be used and enter the name in the Data logging directory field.
  2. Define the Logging filename. This will appear as a prefix in all logging file names. It is recommended that you enter a descriptive name to help you locate a file after it has been logged.
  3. Check that the Raw Logging checkbox is selected.
  4. Enter the Global Logging range in the Start and End fields. This must be entered in lower case and the strings start and end will be accepted in the respective fields to match the length values they represent.
  5. If necessary, select the Limit log file size checkbox and set the Size Limit.
  6. If necessary, select the Sound field logging checkbox. Check with the Person In Charge (PIC) that this information is to be logged.

Graph data can only be logged over the entire monitored section. Whilst it can be run in conjunction with global logging, it can reduce the amount of raw data that can be logged to disk. For more details, refer to your training materials.

  1. Select the Apply and Save buttons. This will stop the controller and restart it with data logging operational. If a property change is made and submitted, a new data log file with creation date/time, incorporated into its file name, will be created when the system restarts.

Dynamic Data Logging

Below is an image of the Dynamic Global Logging dialog:

Dynamic Global Logging dialog

  1. Either enter the text value for the start and end distances in their respective fields.
    OR
    Choose the Select logging area visually button, and choose and then grab the distance measured in the graphical selection.
  2. When the logging interval is defined, select the Off button. It will then change to On, and data logging will commence. To define the logging interval, refer to Step 2 above.
  3. Make sure the data logged does not exceed the maximum write speed of the disk drive that is used. To do this, monitor the write speed on the System Health dashboard. If it approaches the maximum sustained write speed for the disk drive, reduce the amount of data that is logged.
