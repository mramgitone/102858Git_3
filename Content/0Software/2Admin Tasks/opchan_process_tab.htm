FFT Size

Fast Fourier Transform (FFT) is used to perform the spectral analysis. This is where time domain data is transformed into frequency domain data. Short FFT sizes (68 to 128) provide better time domain resolution, but can smear spectral resolution at the expense of temporal time resolution reduction. The trade-off is dependent on the signal the system has to detect. Leaks MUST use longer FFT sizes (256 to 1024), as the time is invariant. Impulsive type disturbances require better time resolution, and a shorter FFT size is best.

Any change can also have an impact on the compute load for a sensor. An increased FFT size will halve the number of detection statistics computed in a given period of time. This provides a reduction in the compute load at the expense of a reduction in temporal resolution.

FFT overlapping percentage

As described above, when the FFT size is increased, time resolution is lost. This can be overcome by using the overlapping FFT option. To do this select a non-zero FFT overlapping percentage. The values for this overlap are 0% (no overlap), 50% (12 of an FFT block overlap) and 75% (34 of an FFT block overlap). This overlap enables a larger FFT size to be chosen without sacrificing temporal resolution.

Example

An FFT size of 1024 with a 50% overlap has the same temporal resolution as an FFT size of 512 with no overlap. And an FFT size of 1024 with a 75% overlap has the same temporal resolution as an FFT size of 256 with no overlap.

However this comes at a cost. An FFT overlap of 50% will require about twice the memory as the same FFT size with no overlap. If an older unit is in use, processing will be slowed down and can stop. This is due to older and slower CUDA cards. It is recommended that you contact Customer Support for a replacement part/unit.

The overlapped FFT is used only in the computation of the stream detection values. It has no effect on raw data or on the default display. As a reminder the FFT overlapping field will be disabled if there is no stream detection value included in the current processing. If you see this field disabled check that you have actually defined a stream and that the stream's alarm report has been selected in the Alarm Report tab.

The Frame Stacking Factor can be used to reduce the processing load to make it easier to process longer s or use higher PRF values! Prior to a processed flow, an average number for the power spectra is calculated. This can smear out the temporal resolution in a similar fashion to when the FFT Size is increased. However, this occurs without the benefit of an increased frequency resolution. The higher the Frame stacking factor, the lower the systems temporal resolution will be. Update Rate

The update rate field specifies how quickly the shots will be sent to the browser. A higher update rate will give you a faster display (provided the drawing can keep up) but may cause the browser to get bogged down. Decreasing the update rate can make the browser more responsive by the reduction of time taken to display the data. If the browser starts to warn you that shots are getting dropped, that is an indicator that you may need to either reduce the update rate or increase the decimation (see below).

 

When the alarm detection properties are tuned, the update rate must be set to the maximum possible value. This is to make sure what is displayed is not misinterpreted. If the browser cannot handle the full update rate,for instance,. if it complains about dropped shots, reduce the monitored range in order to concentrate on the portion of the that is actually tuned, rather than change the update rate to a reduced value.

 

The update rate only affects the display of the data. It does not affect the logged raw data or the logged graph which always record the full data set.

 

The Update Rate field MUST BE set at the maximum rate when the Alarm detection threshold properties are tuned. This provides the maximum time resolution in the Graph display, and is only done in the Playback Mode, if a Playback Speed multiplier of less than one is used. The Sound Field Update rate can be set higher if only a few data samples are being sent to .

The maximum update rate is displayed underneath the Update rate field. If you set this field to a larger value, it will default automatically to the calculated maximum value. Refer to your training materials, for more information.

The spatial decimation factor can be used to decimate the shots before they are sent to be displayed in the browser. For example, a 40km will have over 59000 data samples in each shot whereas most displays are only a couple thousand pixels across. Even the latest top of the line high resolution displays are only about 4000 pixels across. This implies that over 14 data samples are being drawn into a single pixel. The spatial decimation factor allows that decimation to take place on the unit before the data is sent to the browser. This can make the browser significantly more responsive.

 

The spatial decimal factor only affects the live display. It does not affect the processing, or the raw recorded data, or even the graph recorded data. It is strictly a display optimization.

The spatial decimation factor will typically not change the way your display looks when an entire is viewed. However, if you zoom-in, the decimation will start to become obvious. Best practice is to leave out the decimation when you know you are going to zoom into a concentrated , small portion of the . Increase the decimation when you are happy with your configuration and just want to monitor the whole for an extended period of time.
